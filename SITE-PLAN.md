# 网站定位与信息架构定稿

## 1. 网站给谁看

- 首先给佳乐自己看，用来整理、记录和回看成长
- 也给老师、同学、朋友以及其他想了解佳乐的人看

## 2. 访客应该记住什么

> 佳乐是一个想认真生活、持续成长，并愿意把努力变成真实成果的人。

## 3. 网站长期记录什么

1. 自我介绍
2. 座右铭
3. 成长记录
4. 方法记录
5. 生活碎片与联系方式

## 4. 网站核心叙事

```text
首页建立印象
  -> 关于我：我是谁、相信什么
  -> 成长记录：我正在怎样行动，又训练了什么能力
  -> 方法记录：行动里沉淀了哪些可复用方法
  -> 生活碎片：项目之外，我怎样生活
```

## 5. 内容边界

- 不做空洞的炫技页面
- 项目内容按真实进度更新
- 不把所有内容塞在首页
- 不为了显得完整而保留没有内容的栏目
- 不追求一次完成，接受网站随着佳乐一起变化

## 6. 当前版本完成标准

- 每个栏目有独立、明确的页面职责
- 导航能够在所有页面之间往返
- GitHub 和邮箱可以直接访问
- 桌面与手机均可正常阅读
- 项目页能看出真实进展，方法页能留下可复用经验
